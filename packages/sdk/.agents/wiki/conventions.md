# 包内规范

> 此文件由 agents-docs-index skill 管理。
> 以下规范覆盖根级 conventions

## 代码风格

- 遵循 TypeScript 最佳实践
- 文件使用 camelCase 命名
- 函数和变量使用 camelCase 命名
- 常量使用 UPPER_SNAKE_CASE 命名
- 接口使用 PascalCase 命名

## 模块规范

- 每个功能模块单独放在一个文件中
- 模块间通过 index.ts 导出
- 保持模块的单一职责
- 模块间依赖关系清晰

## 类型规范

- 为所有函数和变量添加类型注解
- 使用接口定义复杂类型
- 保持类型定义的一致性
- 避免使用 any 类型

## 测试规范

- 为所有功能编写测试用例
- 使用 Jest 进行单元测试
- 测试覆盖主要功能和边界情况
- 测试代码与源代码分离

## 文档规范

- 为所有公共 API 添加 JSDoc 注释
- 保持 README.md 的更新
- 提供清晰的使用示例
- 文档应包含功能说明和使用方法
