# 包内规范

> 此文件由 agents-docs-index skill 管理。
> 以下规范覆盖根级 conventions

## 代码风格

- 遵循 TypeScript 最佳实践
- 文件使用 camelCase 命名
- 函数和变量使用 camelCase 命名
- 常量使用 UPPER_SNAKE_CASE 命名

## 测试规范

- 为所有功能编写测试用例
- 使用 Jest 进行单元测试
- 测试覆盖主要功能和边界情况
- 测试代码与源代码分离

## 实验规范

- 实验性代码应有明确的注释
- 实验功能应有文档说明
- 保持实验代码的清晰和可读性
- 及时清理不再需要的实验代码

## 文档规范

- 为主要功能提供文档
- 保持 README.md 的更新
- 实验功能应有单独的文档说明
