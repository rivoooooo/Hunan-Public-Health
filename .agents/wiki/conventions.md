# 全局编码规范

> 此文件由 agents-docs-index skill 管理。

## 代码风格

- 使用 TypeScript 进行类型检查
- 遵循项目的 ESLint 和 Prettier 配置
- 保持代码简洁明了，避免过度复杂的逻辑

## 命名约定

- 文件名使用小写字母，单词之间用连字符分隔
- 组件名使用 PascalCase
- 变量和函数名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

## 导入规范

- 优先使用相对路径导入
- 导入顺序：内置模块、第三方模块、本地模块
- 使用明确的导入名称，避免使用默认导入

## 错误处理

- 使用 try-catch 捕获异常
- 对错误进行适当的日志记录
- 向用户提供清晰的错误信息

## 注释规范

- 为复杂的逻辑添加注释
- 使用 JSDoc 格式为函数和类添加文档
- 保持注释与代码同步
